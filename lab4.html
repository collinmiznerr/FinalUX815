Markup & Style II

Took a one-page portfolio site from wireframe to live deployment. Designing the full comp in Figma first, then building it out in HTML and CSS. The layout features a full-width header, a hero text block, and a two-column image and bio section.

Live Preview

Open Lab

Skills Used

Figma HTML Codespaces Semantic elements Two-column Layout

Key Takeaways

Designing in Figma first makes the HTML/CSS build faster. Layout and color decisions are already made.

Semantic elements like header, section, and main improve both accessibility and readability.

Process

1

Built the full design comp in Figma with copywriting and a color palette from Coolors.

2

Created a new repo from the boilerplate and opened the project in Codespaces.

3

Wrote semantic HTML markup following the wireframe structure exactly.

4

Styled with CSS color palette, typography, two-column section, and spacing.

Challenges

Translating the two-column layout from Figma into CSS had some research into how to align an image and text block side by side. I experimented with display: flex to get the columns to behave correctly.
